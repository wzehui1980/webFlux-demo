🔧 核心技术对比

WebFlux vs Spring MVC

特性 WebFlux Spring MVC
编程模型 响应式（异步非阻塞） 命令式（同步阻塞）
容器 Netty（默认） Tomcat（默认）
并发能力 高（少量线程处理大量请求） 中（需要线程池）
适用场景 IO 密集型、高并发 CPU 密集型、传统应用

R2DBC vs JPA

特性 R2DBC JPA/Hibernate
数据库访问 响应式（非阻塞） 阻塞式
性能 高 一般
懒加载 ❌ 不支持 ✅ 支持
生态系统 发展中 成熟

💻 代码示例

1. 创建响应式 REST 接口

@RestController
@RequestMapping("/api/users")
public class UserController {
    
    private final UserService userService;
    
    // 查询所有用户 - 返回 Flux
    @GetMapping
    public Flux<UserEntity> findAll() {
        return userService.findAll();
    }
    
    // 根据ID查询 - 返回 Mono
    @GetMapping("/{id}")
    public Mono<ResponseEntity<UserEntity>> findById(@PathVariable Long id) {
        return userService.findById(id)
                .map(ResponseEntity::ok)
                .defaultIfEmpty(ResponseEntity.notFound().build());
    }
}

2. R2DBC 响应式数据库操作

@Repository
public interface UserRepository extends ReactiveCrudRepository<UserEntity, Long> {
    
    // 根据手机号查询
    Mono<UserEntity> findByPhone(String phone);
    
    // 根据姓名查询
    Mono<UserEntity> findByName(String name);
    
    // 自定义查询
    @Query("SELECT * FROM users WHERE active = :active")
    Flux<UserEntity> findByActive(Boolean active);
}

3. WebClient 响应式客户端调用

@Component
public class UserClient {
    
    private final WebClient webClient;
    
    public Flux<UserEntity> findAllUsers() {
        return webClient.get()
                .uri("/api/users")
                .retrieve()
                .bodyToFlux(UserEntity.class);
    }
    
    public Mono<UserEntity> saveUser(UserEntity user) {
        return webClient.post()
                .uri("/api/users")
                .bodyValue(user)
                .retrieve()
                .bodyToMono(UserEntity.class);
    }
}

❓ 常见问题

Q: 什么时候应该使用 WebFlux？

A: 当你的应用是 IO 密集型（如大量数据库查询、HTTP 调用）、需要高并发处理能力、或者需要实时数据推送（SSE/WebSocket）时，WebFlux 是很好的选择。

Q: R2DBC 和 JPA 可以一起使用吗？

A: 可以，但不推荐。如果必须混用，需要使用 Mono.fromCallable() 包装 JPA 的阻塞调用，在独立的线程池中执行。

Q: 如何处理响应式编程中的错误？

A: 使用 onErrorResume()、onErrorReturn()、onErrorMap() 等操作符进行错误处理和恢复。

Q: WebFlux 的性能一定比 Spring MVC 好吗？

A: 不一定。在高并发 IO 密集型场景下 WebFlux 优势明显，但在 CPU 密集型或低并发场景下，Spring MVC 可能更简单高效。
