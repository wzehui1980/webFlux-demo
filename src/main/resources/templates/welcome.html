💡 关键技术点

1. Mono vs Flux

  • Mono<T>：表示 0 或 1 个元素的异步序列
  • Flux<T>：表示 0 到 N 个元素的异步序列
  • 类比：Mono 类似 Optional，Flux 类似 Stream

2. map vs flatMap

  • map()：同步转换，适用于普通对象转换
  • flatMap()：异步转换，适用于返回 Mono/Flux 的操作

3. WebFlux Security

  • 使用 @EnableWebFluxSecurity 而非 @EnableWebSecurity
  • 实现 ReactiveUserDetailsService 而非 UserDetailsService
  • 返回 Mono<UserDetails> 而非 UserDetails

4. JPA 多对多关系

  • 使用 @ManyToMany 注解建立关联
  • 通过 @JoinTable 指定中间表
  • 手动实现 equals/hashCode 避免懒加载异常

🔧 常见问题

Q1: WebFlux 和 Spring MVC 有什么区别？

A: WebFlux 是响应式的、非阻塞的，适合高并发 I/O 密集型场景；Spring MVC 是传统的 Servlet 模型，阻塞式的。WebFlux 使用 Netty 容器，而 Spring MVC 通常使用 Tomcat。

Q2: 为什么要使用 Netty 而不是 Tomcat？

A: Netty 是响应式的非阻塞 I/O 服务器，与 WebFlux 的响应式编程模型更匹配，性能更好，能更好地发挥 WebFlux 的优势。

Q3: 如何处理阻塞操作（如 JPA）？

A: 使用 Mono.fromCallable() 将阻塞操作包装为响应式类型，在独立的线程池中执行，避免阻塞主事件循环。
